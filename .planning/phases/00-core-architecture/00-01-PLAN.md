# Plan 00-01: Schema & Model Architecture (B2C + C2C)

## Goal
Establish the foundational database schema and Eloquent models for the hybrid Nike B2C (Store) and C2C (Marketplace) platform. This includes user roles, product categories, store products with variants, and marketplace listings.

## Requirements Covered
- **AUTH-01/02**: Foundation for users and roles.
- **PROD-01**: Categories and Products schema.
- **C2C-01**: Marketplace listings schema.

## Proposed Changes

### Database Schema
- **Users**: Add `role` column (enum: customer, seller, admin).
- **Categories**: `uuid`, `name`, `slug`, `parent_id` (nested categories).
- **Products (B2C)**: `uuid`, `category_id`, `name`, `slug`, `description`, `price`, `status`.
- **Product Variants (B2C)**: `uuid`, `product_id`, `sku`, `size`, `color`, `stock`, `price_override`.
- **Marketplace Listings (C2C)**: `uuid`, `user_id`, `product_id` (optional reference to original), `name`, `description`, `price`, `condition_rating` (0-100), `status`.

### Models
- **User**: Add roles and relationships to listings.
- **Category**: Nested relationships.
- **Product**: Relationships to variants and categories.
- **ProductVariant**: Belonging to product.
- **MarketplaceListing**: Belonging to user, optionally referencing a product.

## Execution Steps
1. Create migrations for `categories`, `products`, `product_variants`, and `marketplace_listings`.
2. Update `users` migration or create a new one to add roles.
3. Generate Models for each table with UUID trait and relationships.
4. Run migrations.

## Verification Plan
1. Check database schema via `php artisan db:show` or similar.
2. Verify relationships in Tinker.
