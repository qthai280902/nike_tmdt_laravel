# Phase 4 Plan: Identity & Security

## Backend: Unified Authentication System
- **AuthService**: Implement `login`, `register`, and `logout` logic.
- **Cart Sync Strategy (Critical)**:
  - Inside the `login` process, check if `session('nike_cart')` has items.
  - If yes, merge them into the user's permanent cart (or just keep them in session but associated with the user ID if moving to DB cart later). For now, we will persist the session cart but ensure it remains attached during login.
- **Middleware**:
  - `EnsureUserHasRole`: Check `$user->role === $role`.

## Frontend: Nike Radical Minimalist Auth
- **Login View**: Full-height split screen (Image / Form) or minimalist center-card.
  - Large uppercase typography for "Login".
  - High-precision input fields (no borders except focus).
- **Register View**: Clean multi-step or single-column form.
- **Navbar Update**: Dynamic toggling of "Join Us / Sign In" vs "Account / Logout".
